## Dasar Teori

Xinu (Xinu Is Not Unix) merupakan sistem operasi kecil yang dikembangkan untuk tujuan pembelajaran dan penelitian dalam bidang sistem operasi. Xinu dirancang agar mahasiswa dapat memahami konsep dasar sistem operasi secara lebih mudah karena struktur kode sumbernya lebih sederhana dibandingkan sistem operasi modern seperti Linux atau Windows.

Sistem operasi sendiri merupakan perangkat lunak yang berfungsi sebagai penghubung antara pengguna dengan perangkat keras komputer. Sistem operasi mengatur berbagai sumber daya komputer seperti prosesor, memori, serta perangkat input dan output agar dapat digunakan oleh program aplikasi secara efisien. Tanpa adanya sistem operasi, perangkat keras komputer tidak dapat digunakan secara optimal karena tidak ada sistem yang mengatur komunikasi antar komponen.

Xinu memiliki beberapa komponen penting yang digunakan untuk mempelajari konsep sistem operasi, seperti manajemen proses, penjadwalan CPU, komunikasi antar proses, serta pengelolaan memori. Dengan mempelajari Xinu, mahasiswa dapat memahami bagaimana sistem operasi bekerja pada level yang lebih rendah dan bagaimana proses eksekusi program dapat diatur oleh sistem.

Dalam praktikum ini, Xinu dijalankan menggunakan Virtual Machine melalui aplikasi VirtualBox. Virtual Machine memungkinkan pengguna untuk menjalankan sistem operasi tambahan di dalam sistem operasi utama tanpa perlu melakukan instalasi langsung pada perangkat keras. Hal ini sangat membantu dalam proses pembelajaran karena konfigurasi yang dilakukan tidak akan merusak sistem utama pada komputer.

Melalui proses instalasi dan konfigurasi Xinu pada VirtualBox, praktikan dapat memahami bagaimana lingkungan pengembangan sistem operasi disiapkan serta bagaimana konfigurasi mesin virtual dilakukan agar sistem dapat berjalan dengan baik.

# Guided Praktikum

## 1. Import Development System VM

Langkah pertama adalah mengimport virtual machine yang telah disediakan pada modul praktikum. File VM memiliki format `.ova` yang berisi konfigurasi sistem yang sudah disiapkan untuk proses pengembangan Xinu.

Langkah-langkahnya:

1. Buka aplikasi **VirtualBox**.
2. Pilih menu **File → Import Appliance**.
3. Pilih file `development-system.ova`.
4. Klik **Next**.
5. Tunggu hingga proses import selesai.
6. Setelah selesai, VM **development-system** akan muncul pada daftar VirtualBox.

---

## 2. Konfigurasi Serial Port Development System

Setelah virtual machine berhasil diimport, langkah berikutnya adalah melakukan konfigurasi pada bagian Serial Port.

Langkah-langkahnya:

1. Pilih VM **development-system**.
2. Klik **Settings**.
3. Masuk ke menu **Serial Ports**.
4. Aktifkan **Enable Serial Port**.
5. Pastikan opsi **Connect to existing pipe/socket** tidak dicentang.

<img src="images/2-4.png" width="600">

---

## 3. Konfigurasi Display

Pengaturan display dilakukan agar tampilan virtual machine dapat berjalan dengan baik.

Langkah-langkahnya:

1. Masuk ke **Settings** pada VM.
2. Pilih menu **Display**.
3. Atur **Video Memory** sesuai kebutuhan.
4. Gunakan **Graphics Controller: VMSVGA**.
5. Aktifkan **Enable 3D Acceleration**.

<img src="images/2-5.png" width="600">

---

## 4. Import Backend VM

Setelah melakukan konfigurasi pada development-system, langkah berikutnya adalah mengimport virtual machine **backend** yang akan digunakan sebagai sistem target Xinu.

Langkah-langkahnya:

1. Buka **VirtualBox**.
2. Pilih **File → Import Appliance**.
3. Pilih file `backend.ova`.
4. Klik **Next**.
5. Tunggu hingga proses import selesai.
6. VM **backend** akan muncul pada daftar mesin virtual.

<img src="images/2-6.png" width="600">

---

## 5. Konfigurasi Network Backend (Adapter 1)

Langkah berikutnya adalah melakukan konfigurasi jaringan agar **development-system dan backend dapat saling terhubung**.

Langkah-langkahnya:

1. Pilih VM **backend**.
2. Klik **Settings**.
3. Pilih menu **Network**.
4. Pada **Adapter 1** lakukan pengaturan:

- Centang **Enable Network Adapter**
- Attached to : **Internal Network**
- Name : **xinu_net**

Konfigurasi ini memungkinkan kedua virtual machine berkomunikasi melalui jaringan internal VirtualBox.

<img src="images/2-7.png" width="600">

---

## 6. Konfigurasi Network Backend (Adapter 2)

Selain Adapter 1, konfigurasi juga dilakukan pada Adapter 2.

Langkah-langkahnya:

1. Masuk ke menu **Network** pada VM **backend**.
2. Pilih **Adapter 2**.
3. Centang **Enable Network Adapter**.
4. Pilih **Attached to : NAT**.

Konfigurasi NAT memungkinkan virtual machine terhubung ke jaringan internet melalui host.

<img src="images/2-8.png" width="600">

---

## 7. Konfigurasi Serial Port Backend

Langkah berikutnya adalah melakukan konfigurasi Serial Port pada backend agar dapat berkomunikasi dengan development-system.

Langkah-langkahnya:

1. Masuk ke **Settings** VM **backend**.
2. Pilih menu **Serial Ports**.
3. Aktifkan **Enable Serial Port**.
4. Gunakan pengaturan berikut:

- **Port Number : COM1**
- **Port Mode : Host Pipe**
- Centang **Connect to existing pipe/socket**

5. Pada bagian **Path/Address**, gunakan path yang sama seperti yang digunakan pada development-system.

Konfigurasi ini memungkinkan kedua VM berkomunikasi melalui pipe/socket.

# Kesimpulan

Berdasarkan praktikum yang telah dilakukan, proses instalasi Xinu pada VirtualBox memerlukan beberapa tahapan penting seperti import virtual machine, konfigurasi jaringan, serta pengaturan serial port. Dua virtual machine yang digunakan yaitu **development-system** dan **backend** memiliki fungsi yang berbeda namun saling terhubung dalam proses pengembangan sistem operasi Xinu.

Virtual machine development-system digunakan sebagai lingkungan pengembangan untuk melakukan kompilasi program, sedangkan backend digunakan sebagai sistem target tempat Xinu dijalankan. Dengan konfigurasi jaringan internal dan serial port yang tepat, kedua mesin virtual dapat saling berkomunikasi dengan baik.

Melalui praktikum ini, praktikan dapat memahami bagaimana lingkungan pengembangan sistem operasi disiapkan serta bagaimana virtualisasi digunakan untuk mempermudah proses pembelajaran sistem operasi tanpa mempengaruhi sistem utama pada komputer.
